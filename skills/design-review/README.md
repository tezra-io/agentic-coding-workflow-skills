# design-review

Architecture-first kickoff workflow for new projects or major features.

## Output
- Approved design doc (`docs/<issue-id>-design.md`)
- Project `CLAUDE.md`
- Linear implementation issues with dependency-aware priority ordering

## Best for
- New modules/systems
- Multi-file architectural changes
- Work requiring explicit trade-off decisions

See `SKILL.md` for full workflow and review criteria.
